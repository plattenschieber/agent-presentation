# AGENTS.md

Guide for AI agents working in this repository.

## What This Is

A [Slidev](https://sli.dev) presentation repo for the Agents Anonymous meetup in Cologne (formerly Claude Code Anonymous). Independent slide decks built to static HTML and deployed to GitHub Pages.

## Project Structure

```
presentations/       # Slidev markdown slide decks (the source of truth)
scripts/             # Build scripts (create-index.js generates the landing page)
public/              # Shared images and assets
dist/                # Build output (gitignored, generated by build)
.github/workflows/   # GitHub Pages deploy pipeline
```

## Building

```sh
npm ci               # Install dependencies (use ci, not install)
npm run build        # Build everything (all presentations + index page)
npm run build:round1 # Build a single presentation
npm run dev:round4   # Live dev server for a single presentation
```

### Build Verification

There are no automated tests. After any change, verify by building:

```sh
npm run build
```

A successful build produces `dist/` with subdirectories: `round1/`, `round2/`, `round3/`, `round4/`, `ast-grep/`, `lessons/`, `round6/`, and an `index.html`.

**Check the build succeeded** by confirming these files exist:
- `dist/index.html`
- `dist/round1/index.html`
- `dist/round2/index.html`
- `dist/round3/index.html`
- `dist/round4/index.html`
- `dist/ast-grep/index.html`
- `dist/lessons/index.html`
- `dist/round6/index.html`

## Common Mistakes

### 1. Getting `--base` and `--out` paths wrong

Each presentation builds with a `--base` (URL path prefix for GitHub Pages) and `--out` (filesystem output directory). These are different things and both are relative:

- `--base` must match the GitHub Pages URL: `/agent-presentation/<name>/`
- `--out` is relative to the **presentation file's directory** (`presentations/`), so it's `../dist/<name>`

Past bugs fixed in this repo were exactly this: wrong `--out` paths. If you add a new presentation, follow the existing pattern in `package.json` exactly.

### 2. Forgetting to update `scripts/create-index.js`

The landing page is **hardcoded HTML** in `scripts/create-index.js`. Adding a new presentation requires:
1. A new `build:<name>` script in `package.json`
2. Adding it to the `build:all` chain in `package.json`
3. A new card in `scripts/create-index.js`
4. Optionally a `dev:<name>` script

### 3. Forgetting `.nojekyll`

GitHub Pages needs a `.nojekyll` file in `dist/` to serve files correctly. The `create-index.js` script handles this. Don't delete it or bypass the index build step.

### 4. Editing `dist/` directly

Everything in `dist/` is generated. Never edit files there; they'll be overwritten on the next build.

## Adding a New Presentation

1. Create `presentations/<name>.md` using an existing deck as a template
2. Add scripts to `package.json`:
   ```
   "build:<name>": "slidev build presentations/<name>.md --base /agent-presentation/<name>/ --out ../dist/<name>"
   "dev:<name>": "slidev presentations/<name>.md --open"
   ```
3. Append `npm run build:<name>` to the `build:all` chain (before `build:index`)
4. Add a card to `scripts/create-index.js` with title, description, date, and href `./<name>/`
5. Run `npm run build` and verify `dist/<name>/index.html` exists

## Deployment

Automatic on push to `main` via GitHub Actions (`.github/workflows/deploy.yml`). The workflow runs `npm ci && npm run build` and deploys `dist/` to GitHub Pages.

Live site: https://plattenschieber.github.io/agent-presentation/

## Slide Authoring

Slides are Markdown with Slidev extensions. Key conventions used in this repo:
- `---` separates slides
- Front matter in the first slide sets theme, highlighter, transition
- `<v-click>` for incremental reveals
- Images from `public/` are referenced as `/filename.png` (Slidev serves `public/` at root)

## Dependencies

Minimal on purpose. Only `@slidev/cli` and `@slidev/theme-default`. Don't add dependencies unless absolutely necessary.
